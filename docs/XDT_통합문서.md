## 1. XDT 소개

**XRX Digital Twin Platform**

### XDT: eXtended Digital Twin

> "보는 관제(View)에서 시키는 지휘(Command)로"

공장의 모든 로봇과 설비를 가상 세계에 복제하고, AI가 직접 판단하고 명령하는 — **자율 제조의 운영체제**

---

## 2. 왜 XDT가 필요한가

오늘날 제조 현장에서 로봇은 점점 더 많아지고 있지만, 정작 **로봇을 똑똑하게 만드는 일**은 여전히 느리고 위험합니다. 새로운 동작을 가르치려면 실제 로봇으로 수백 번 시험해야 하고, 시험 중 충돌이나 오작동이 발생하면 수천만 원의 장비가 손상됩니다. 하나의 공정을 바꾸는 데 몇 주가 걸리고, 그 사이 라인은 멈춰 있습니다.

기존 '디지털 트윈'이라 불리는 솔루션들은 대부분 **3D 화면으로 현장을 '보여주는' 수준**에 머물러 있습니다. 사람이 데이터를 읽고, 사람이 판단하고, 사람이 명령해야 합니다. 결국 모니터링 대시보드에 불과한 것입니다.

> "공장에 로봇이 100대 있는데, 왜 여전히 사람이 하나하나 관리해야 할까?"
> XDT는 바로 이 질문에서 출발합니다.

---

## 3. XDT는 무엇인가

XDT(eXtended Digital Twin)는 **공장 전체를 가상 세계에 그대로 복제**하고, 그 안에서 AI가 스스로 생각하고, 판단하고, 로봇에게 직접 명령을 내리는 **'자율 제조의 운영체제(Virtual OS)'**입니다.

쉽게 비유하자면 이렇습니다. 기존 디지털 트윈이 **CCTV**라면, XDT는 **AI 사령관**입니다. CCTV는 상황을 보여주기만 하지만, AI 사령관은 상황을 파악하고 최적의 작전을 짜서 부대(로봇)에게 직접 명령합니다. 사람은 전략만 지시하면, 나머지는 XDT가 알아서 처리합니다.

또 하나의 비유를 들자면, **RTS(실시간 전략) 게임**과 같습니다. 게임에서 플레이어가 유닛들에게 "이 지점으로 이동", "이 자원을 채취"라고 명령하면 유닛들이 알아서 움직이듯, XDT에서는 운영자가 "A라인 생산량 증가"라고 말하면 AI가 로봇 배치, 동선, 속도를 자동으로 최적화합니다.

---

## 4. 왜 가치 있는가

**첫째, 물리적 시행착오가 사라집니다.** XDT의 시뮬레이터 안에서는 로봇이 아무리 충돌하고 넘어져도 장비 손상이 없습니다. 새로운 공정, 새로운 동작을 가상 세계에서 충분히 테스트한 뒤, 검증된 것만 실제 로봇에 적용합니다. 현장 리스크를 획기적으로 줄일 수 있습니다.

**둘째, 로봇이 스스로 똑똑해집니다.** XDT 안의 AI는 시뮬레이션 데이터를 끊임없이 학습합니다. 처음에는 사람이 시범을 보여주지만, 그 경험 데이터를 바탕으로 AI가 자기 스스로 더 나은 방법을 찾아갑니다. 마치 바둑 AI가 프로 기사의 기보를 학습한 뒤 스스로 더 강해지는 것과 같은 원리입니다.

**셋째, 로봇 하드웨어 원가를 절감할 수 있습니다.** 보통 로봇에는 비싼 컴퓨터(두뇌)가 탑재되어 있습니다. XDT는 이 두뇌를 서버(클라우드)로 옮기고, 현장 로봇에는 센서와 모터만 남깁니다. 이른바 'Brainless Robot' 전략으로, 로봇 한 대당 하드웨어 원가를 절감할 수 있습니다.

**넷째, 어떤 로봇이든 연결됩니다.** 제조 현장에는 여러 회사의 장비가 섞여 있습니다. 보통은 A사 로봇과 B사 센서가 서로 말이 통하지 않습니다. XDT는 자체 통신 표준인 NSS(Node Signal Specification)를 통해 제조사와 관계없이 모든 장비를 하나의 네트워크로 연결합니다.

## 5. 핵심 서비스 기능

XDT 플랫폼은 6개의 핵심 모듈이 유기적으로 연결되어, 명령부터 학습까지 하나의 순환 고리를 형성합니다.

### xDT.Play — 게임처럼 조종하는 관제 화면
복잡한 그래프 대신, RTS 게임처럼 직관적인 인터페이스로 로봇을 조종합니다. 운영자가 직접 조작한 데이터는 AI 학습의 핵심 재료(기보)가 됩니다.

### xDT.LLM — 말로 명령하는 AI 통역사
"A라인 속도 올려줘"처럼 자연어로 말하면, AI가 시스템 명령어로 자동 변환합니다. 코딩을 몰라도 누구나 로봇을 제어할 수 있습니다.

### xDT.AgentX — 로봇의 두뇌
드론, 4족 보행 로봇, 로봇팔 등 어떤 형태든 상관없이 하나의 표준으로 제어합니다. '인지 → 판단 → 실행'을 자율적으로 수행하는 엣지 AI 레이어입니다.

### xDT.SpaceTime — 중앙 관제탑
수십, 수백 대의 로봇 상태를 실시간으로 파악하고 군집 단위로 지휘합니다. 모든 데이터의 시계열 기록과 3D 시각화를 담당하는 플랫폼의 심장입니다.

### xDT.Sim — 리스크 제로 시뮬레이터
Google DeepMind의 MuJoCo 물리엔진으로 현실과 동일한 물리 법칙을 재현합니다. 가상 세계에서 무한 반복 훈련 후, 검증된 결과만 현실에 적용합니다.

### xDT.MindX — 스스로 진화하는 AI
시뮬레이션과 현장 데이터를 끊임없이 학습하여, 점점 더 효율적인 운영 전략을 스스로 만들어냅니다. 바둑 AI처럼 경험이 쌓일수록 강해집니다.

## 6. 작동 원리: Train → Deploy → Optimize

XDT의 모든 가치는 이 세 단계의 무한 반복에서 나옵니다.

| 단계 | 이름 | 설명 |
|------|------|------|
| ① | **Train** (가상 훈련) | 가상 공간에서 AI와 로봇을 반복 훈련. 실패해도 장비 손상 없이 안전하게 실험 |
| ② | **Deploy** (실전 배포) | 검증이 완료된 AI 모델과 동작만 실제 로봇에 적용. 첫 시도부터 성공하는 배포 |
| ③ | **Optimize** (지속 최적화) | 현장에서 수집된 실제 데이터가 다시 가상 세계로 돌아가 AI를 더 강하게 만듦 |

### XDT SOLUTION 전체 흐름

```
xDT.Play (명령 입력)
    ↓ COMMANDS
xDT.AgentX (AI 판단 → 로봇 행동 지시)
    ↓ EXECUTE
xDT.Sim (가상 환경 시뮬레이션 및 검증)
    ↓ SIMULATE

명령 → 실행 → 검증 → 학습 → 최적화 순환

xDT.LLM (자연어 → 시스템 명령 변환)     TRANSLATE
xDT.SpaceTime (전체 상태 기록 및 시각화)  MONITOR
xDT.MindX (데이터 학습 → 스스로 진화)     LEARN
```

## 7. 기존 디지털 트윈 vs XDT 비교

XDT는 '보여주는 도구'가 아니라 '실행하는 플랫폼'입니다.

| 구분 | 일반적인 디지털 트윈 | XRX XDT Solution |
|------|----------------|-----------------|
| **정체성** | 3D 뷰어 & 모니터링 대시보드 | 자율 운영 OS & AI 지휘관 |
| **제어 방식** | 수동 관리 (사람이 판단) | 능동 제어 (AI가 판단 및 실행) |
| **연결성** | 제조사별 폐쇄적 연결 | NSS 기반 초연결 (Any Vendor) |
| **사용자 경험** | 복잡한 그래프와 수치 | RTS 게임형 직관적 조작 |
| **개발 속도** | 물리 테스트 위주 (느림) | 가상 검증 위주 (획기적으로 빠름) |
| **리스크** | 현장 사고 위험 존재 | 가상 시뮬레이션으로 현장 리스크 최소화 |
| **로봇 비용** | 고가의 자체 컴퓨팅 필수 | Brainless Robot으로 원가 절감 가능 |
| **확장성** | 소수 장비 개별 관리 | 대규모 군집 제어 가능 |

## 9. 핵심 기술 상세

### 핵심 기술 01: xDT.AgentX — 단일 로봇 실행 플랫폼

4족 보행 로봇, 드론, 로봇 팔 등 형태에 상관 없이 xDT.AgentX가 표준 명령어로 변환하여 장치를 제어합니다.

```
Command Center (명령 센터) → xDT.AgentX (인지 판단 제어) → Real & Sim (실제 & 가상 환경)
```

- **1 Robot = 1 AgentX**: 개별 로봇마다 독립된 AgentX 인스턴스
- **자율성**: 중앙 서버 연결이 끊겨도 독자적인 생존과 임무 수행 가능
- **동일성**: 가상(Sim)과 현실(Real)에서 완벽히 동일한 로직으로 작동

### 핵심 기술 02: xDT.SpaceTime — 중앙 관제탑

디지털트윈의 '중앙 두뇌'이자 '통합 관제탑'. 여러 장치를 마치 게임 유닛처럼 다룹니다.

- **군집 제어** (Orchestrates Groups): 수십 대의 이기종 로봇을 통합 지휘
- **임무 기반 제어** (Mission-Based): 작업 목표에 따라 최적의 로봇에게 자동 분배
- **중앙 지능** (Central Intelligence): 전체 시스템 상태를 기반으로 전략적 판단

### 핵심 기술 03: xDT.MindX — 지능/학습 모듈

xDT.Spacetime으로부터 정보를 받아 AI가 스스로 학습하는 모듈

```
xDT.Spacetime (데이터) → xDT.MindX (지능/학습) → Insights & Knowledge
```

### 핵심 기술 04: xDT.Sim — 시뮬레이터

학습하고 실행하고 분석하고 개선하고를 반복합니다.

| 단계 | 이름 | 설명 |
|------|------|------|
| STEP 1 | Virtual Training | 가상훈련 |
| STEP 2 | Real World | 실제 배포 |
| STEP 3 | Live Feedback | 실시간 피드백 |
| STEP 4 | Improvement | 지속적 개선 |

### 핵심 기술 05: xDT.Play — 사용자 인터페이스

사용자가 디지털트윈과 상호작용하는 최종 화면입니다.

```
User Input (사용자 명령) → xDT.LLM (DSL 변환) → Virtual World (가상 세계 확인)
```

### 핵심 기술 06: xDT.LLM — 자연어 변환 엔진

사용자와 디지털 트윈 간의 통역사 역할을 합니다. (OpenAI, Google AI, Meta 멀티 프로바이더 지원)

```
User Language (자연어) → AI Powered (DSL 변환) → Twin Command (트윈 명령어)
```

## 10. 기술 스택

### 물리엔진: NVIDIA PhysX → Google DeepMind MuJoCo

Isaac Sim은 PhysX 기반으로 '안정적'일 뿐, 결정론적이지 않습니다. 학습 · GT 생성 · 검증에서는 결정론이 핵심이고, 그 역할은 MuJoCo가 압도적으로 우수합니다.

> **결정론(Determinism)**: 같은 초기 상태와 같은 입력이면, 언제 · 어디서 실행해도 항상 같은 결과가 나온다는 성질.

**MuJoCo**로 결정론적 물리 · GT · 학습을 담당하고, **Unreal Engine 5**로 실시간 시각화 · 인터랙션 · 데모를 담당하는 **분리 구조**가 가장 합리적입니다.

| 엔진 | 역할 |
|------|------|
| **MuJoCo** | 결정론적 물리, GT(Ground Truth), 강화학습 훈련, 제어 검증 |
| **Unreal Engine 5** | 실시간 시각화, 인터랙션, 데모 시연, 사용자 경험 |

#### MuJoCo 분야별 사용 이유

| 분야 | MuJoCo 사용 이유 |
|------|---------------|
| 강화학습 (RL) | 결정론 + 속도 |
| MPC / 최적제어 | 정확한 dynamics |
| 전통 제어 | inverse dynamics |
| 시스템 식별 | 재현성 |
| 로봇 설계 | 구조 실험 |
| 하이브리드 제어 | 제어+학습 통합 |

### XDT.Node.xDT.AgentX 소프트웨어 스택

```
┌─────────────────────────────────────────────────────┐
```

### 3가지 핵심 특징

1. **로봇의 '두뇌' 역할**: 각 로봇에 탑재되어 자율주행과 자동화 제어를 담당. 상위 시스템 연결 없이도 독립적으로 로봇을 제어 가능.
2. **뛰어난 호환성과 연결성**: NSS & ROS 2 기반으로 다양한 벤더의 로봇과 센서를 단일 표준으로 연동.
3. **확장 가능한 관제 시스템**: 단일 로봇에서 군집 로봇까지, 규모에 관계없이 유연하게 확장되는 관제 아키텍처.

### NSS (Node Signal Specification)

디지털트윈과 장비 간의 '공용어'. VSS(Vehicle Signal Specification)를 로봇 산업에 맞게 차용한 자체 표준으로, **Vendor Agnostic 확장성**을 제공합니다.

| 특성 | 설명 |
|------|------|
| Vendor Agnostic | 특정 제조사에 종속되지 않는 유연한 확장성 |
| 표준화된 데이터 모델 | 센서 값, 상태 정보, 제어 명령이 일관된 형식으로 정의 |
| AI 친화적 구조 | AI 학습에 바로 사용될 수 있는 구조화된 데이터 |
| ROS 2 호환 | ROS 2 생태계와 자연스럽게 연동 |
| 양방향 통신 | 상태 조회(Read)와 제어 명령(Write)을 통일된 인터페이스로 처리 |

### 하드웨어 전략

#### Brainless Robot Solution
로봇의 두뇌(컴퓨팅)를 디지털 트윈 서버로 이관하여 하드웨어 원가를 절감할 수 있습니다. 현장 로봇은 센서와 액추에이터만 탑재합니다.

#### Retrofit
구형 설비를 NSS 호환 센서로 장착하여 지능형 노드로 업그레이드. 기존 투자를 보호하면서 디지털 트윈 생태계에 편입합니다.
