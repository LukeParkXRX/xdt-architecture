## 1. 개요

XDT(eXtended Digital Twin)는 단순한 현장 모니터링이나 3D 시각화 도구가 아닙니다.

물리 세계(Physical World)와 가상 세계(Virtual World)를 실시간으로 동기화하여, **AI가 현장을 직접 제어하고 미래를 예측하는 '자율 제조의 운영체제(Virtual OS)'**입니다.

```
┌──────────────────────────────────────────────────────────┐
│                   XDT SOLUTION 전체 흐름                    │
│                                                          │
│   DT.Play ──COMMANDS──► DT.AgentX ──EXECUTE──► DT.Sim   │
│     ▲                                            │       │
│     │ Central Hub                    FEEDBACK & MONITOR   │
│     │                                            ▼       │
│   DT.LLM ◄──DATA FEED── DT.Spacetime ◄──OPTIMIZE── DT.MindX │
│                                                          │
│              명령 → 관리 → 실행 → 학습 순환                  │
└──────────────────────────────────────────────────────────┘
```

## 3. 6대 핵심 기술 아키텍처

### 3.1 DT.AgentX — 단일 로봇 실행 플랫폼

> **핵심 기술 01** | "DT.AgentX > 생각하고 > 행동한다"

4족 보행 로봇, 드론, 로봇 팔 등 형태에 상관 없이 DT.AgentX가 표준 명령어로 변환하여 장치를 제어합니다.

#### 작동 구조

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│ Command      │     │   DT.AgentX      │     │   Real & Sim     │
│ Center       │ ──► │                  │ ──► │                  │
│ (명령 센터)   │     │  인지 판단 제어    │     │ 실제 & 가상 환경   │
│              │     │  학습된 AI 모델    │     │ 장비 및 시뮬레이션  │
│ From         │     │  & 추론 및 결정    │     │ 제어              │
│ DT.Spacetime │     │                  │     │                  │
└──────────────┘     └──────────────────┘     └──────────────────┘
```

#### 핵심 원칙

- **1 Robot = 1 AgentX**: 개별 로봇마다 독립된 AgentX 인스턴스
- **자율성**: 중앙 서버 연결이 끊겨도 독자적인 생존과 임무 수행 가능
- **동일성**: 가상(Sim)과 현실(Real)에서 완벽히 동일한 로직으로 작동

#### XDT.Node.DT.AgentX 소프트웨어 스택

> "다양한 언어와 ROS 2를 지원하며, 로봇 한 대의 자율주행부터 여러 대의 통합 관제까지 가능한 만능 로봇 제어 소프트웨어"

```
┌─────────────────────────────────────────────────────┐
│              CLI  /  DigitalTwin Bridge              │
├─────────────────────────────────────────────────────┤
│                      NSS API                        │
├─────────────────────────────────────────────────────┤
│                    NSS Runtime                      │
├─────────────────────────────────────────────────────┤
│          Mission Layer (임무, 인지, 계획, 행동)        │
├─────────────────────────────────────────────────────┤
│                Navigation & SLAM                    │
├─────────────────────────────────────────────────────┤
│   Behavior & Gait Control (보행패턴, 상태머신, 동작조합) │
├─────────────────────────────────────────────────────┤
│   Motion Control Layer (역기구학, 자세제어, 균형제어)    │
├─────────────────────────────────────────────────────┤
│ Embedded Control (PID, 센서, I/O, 모터루프, 토크계산)   │
├────────────────────────┬────────────────────────────┤
│       Real (물리)       │     XDT.Simulator (가상)    │
└────────────────────────┴────────────────────────────┘
```

#### 3가지 핵심 특징

1. **로봇의 '두뇌' 역할 (자율주행 및 제어)**
   - 각 로봇에 탑재되어 자율주행과 자동화 제어를 담당하는 핵심 모듈
   - 상위 시스템 연결 없이도 독립적으로 로봇을 제어 가능
   - 지도 작성, 수정, 게임패드 수동 조작까지 모두 지원

2. **뛰어난 호환성과 연결성 (NSS & ROS 2)**
   - NSS(Node Signal Specification) 자체 통신 규약으로 상태 조회나 설정을 쉽게 수행
   - ROS 2 생태계와 연동되어 확장성이 좋고, C++, Python, Java 등 다양한 언어로 개발 가능

3. **확장 가능한 관제 시스템**
   - XDT.SpaceTime과 연동하면 로봇이 여러 대가 되어도 통합 관제 가능
   - 트래픽 제어, 인프라 연동 등 스케일아웃 지원

### 3.2 DT.SpaceTime — 중앙 관제탑

> **핵심 기술 02** | 디지털트윈의 '중앙 두뇌'이자 '통합 관제탑'

장치들이 존재하는 **"공간(Space)"**과 행동하는 **"시간(Time)"**을 통합 관리합니다.

여러 장치를 마치 게임 유닛처럼 다룹니다. 최적의 경로를 계산하고, 충돌을 피하며, 알맞은 장치에게 자동으로 임무를 할당합니다.

#### 핵심 기능

| 기능 | 설명 |
|------|------|
| **군집 제어** (Orchestrates Groups) | 수십 대의 이기종 로봇을 통합 지휘 |
| **임무 기반 제어** (Mission-Based) | 작업 목표에 따라 최적의 로봇에게 자동 분배 |
| **중앙 지능** (Central Intelligence) | 전체 시스템 상태를 기반으로 전략적 판단 |

#### RTS 게임형 인터페이스

- 드래그-셀렉트로 로봇 그룹 선택
- 우클릭으로 이동/작업 명령
- 순찰(Patrol), 호위(Escort) 등 RTS 게임형 명령 체계

| 기술 | 역할 | 강점 |
|------|------|------|
| **Unreal Engine 5** | 고퀄리티 렌더링 & 시각화 | 포토리얼리스틱 그래픽, 실시간 인터랙션 |
| **MuJoCo** | 결정론적 물리 엔진 | 정확한 물리 시뮬레이션, 재현 가능한 실험 |

#### SIL (Software-In-the-Loop)

실제 로봇에 들어갈 소프트웨어를 가상 환경에서 먼저 검증합니다.
- 물리적 로봇 없이 소프트웨어 동작을 완벽하게 테스트
- 현장 사고를 미리 방지하여 **하드웨어 리스크 0%**
- 개발 속도를 물리 테스트 대비 **10배 이상** 가속

### 3.5 DT.Play — 사용자 인터페이스

> **핵심 기술 05** | "Real Time Feedback: 명령 > 변환 > 확인"

사용자가 디지털트윈과 상호작용하는 최종 화면입니다.

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│ User Input   │     │ DT.LLM:          │     │ DT.Play:         │
│ (사용자 명령)  │ ──► │ Command          │ ──► │ Virtual World    │
│              │     │ Conversion       │     │ Action           │
│ · 언어 입력   │     │ (명령어 변환)     │     │ (가상 세계 확인)  │
```

### 3.6 DT.LLM — 자연어 변환 엔진

> **핵심 기술 06** | "DT LLM > Talk > Think > Act"

사용자와 디지털 트윈 간의 **통역사** 역할을 합니다.

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│ User Language│     │   AI Powered     │     │ Twin Command     │
│ (사용자 언어)  │ ──► │ (DSL 변환)       │ ──► │ (트윈 명령어)     │
│              │     │                  │     │                  │
│ · 언어 입력   │     │ · OpenAI         │     │ 결과 실시간      │
│ · 음성/텍스트 │     │ · Google AI      │     │ 시각화 및 확인    │
│ · AI (LLM)   │     │ · Meta AI        │     │                  │
└──────────────┘     └──────────────────┘     └──────────────────┘
```

#### 변환 파이프라인

1. **Talk** — 사용자의 자연어 입력을 수신하고 의도를 파악
2. **Think** — AI 모델이 명령의 맥락을 분석하고 최적의 DSL 명령어로 변환
3. **Act** — 변환된 명령을 디지털트윈 시스템에 전달하여 실행
4. **Feedback** — 실행 결과를 사용자에게 보고하고, 필요시 명령을 정제

## 4. 물리엔진 전략: PhysX → MuJoCo

### 전환 배경

XDT의 핵심 가치인 **"가상에서의 완벽한 훈련"** 을 실현하기 위해, 물리엔진의 **결정론(Determinism)** 은 타협할 수 없는 핵심 요구사항입니다.

> **결정론(Determinism)이란?**
> 같은 초기 상태와 같은 입력이면, **언제 · 어디서 실행해도 항상 같은 결과가 나온다**는 성질.
> 이것이 보장되지 않으면, 학습 결과의 재현성이 없어지고, GT(Ground Truth) 생성이 불가능합니다.

### PhysX의 한계

- PhysX는 **비결정론적** 물리 엔진
- Isaac Sim은 PhysX 기반으로 "안정적"일 뿐, **결정론적이지 않음**
- 학습·GT 생성·검증에서는 결정론이 핵심이고, 그 역할은 MuJoCo가 압도적으로 우수

### 전환 결론

```
┌─────────────────────────────────────────────────────────────┐
│                  XDT 물리엔진 이원화 전략                      │
│                                                             │
│  ┌─────────────────────┐     ┌─────────────────────────┐   │
│  │      MuJoCo         │     │    Unreal Engine 5      │   │
│  │                     │     │                         │   │
│  │ · 결정론적 물리      │     │ · 실시간 시각화          │   │
│  │ · GT(Ground Truth)  │     │ · 인터랙션              │   │
│  │ · 강화학습 훈련      │     │ · 데모 시연             │   │
│  │ · 제어 검증          │     │ · 사용자 경험           │   │
│  │                     │     │                         │   │
│  │ ══ 학습 & 검증 ══    │     │ ══ 시각화 & UX ══       │   │
│  └─────────────────────┘     └─────────────────────────┘   │
│                                                             │
│ "특히 우리 같이 게임 기술에 익숙한 사람에게는 최고의 선택지"     │
└─────────────────────────────────────────────────────────────┘
```

### MuJoCo 분야별 사용 이유

| 분야 | MuJoCo 사용 이유 | 세부 설명 |
|------|---------------|----------|
| **강화학습 (RL)** | 결정론 + 속도 | 수백만 에피소드를 빠르고 재현 가능하게 실행 |
| **MPC / 최적제어** | 정확한 dynamics | 모델 기반 제어에 필수적인 정밀 물리 모델링 |
| **전통 제어** | Inverse dynamics | 역동역학 계산으로 정확한 토크 제어 가능 |
| **시스템 식별** | 재현성 | 동일 실험을 반복하여 물리 파라미터 정밀 추정 |
| **로봇 설계** | 구조 실험 | 다양한 기구학적 구조를 빠르게 프로토타이핑 |
| **하이브리드 제어** | 제어 + 학습 통합 | 전통 제어와 강화학습을 유기적으로 결합 |

## 5. NSS (Node Signal Specification)

### 개념

NSS는 디지털트윈과 물리 노드 간의 **'공용어(Language)'**입니다.

로봇 제조사마다 제각각인 통신 방식을 하나의 표준으로 통일하고, AI가 데이터를 즉시 이해하고 학습할 수 있도록 설계된 통신 규약입니다.

> **기원**: 자동차 산업의 VSS(Vehicle Signal Specification)를 로봇 산업에 맞게 차용한 개념

### 핵심 특성

| 특성 | 설명 |
|------|------|
| **Vendor Agnostic** | 특정 제조사에 종속되지 않는 유연한 확장성 |
| **표준화된 데이터 모델** | 센서 값, 상태 정보, 제어 명령이 일관된 형식으로 정의 |
| **AI 친화적 구조** | AI 학습에 바로 사용될 수 있는 구조화된 데이터 |
| **ROS 2 호환** | ROS 2 생태계와 자연스럽게 연동 |
| **양방향 통신** | 상태 조회(Read)와 제어 명령(Write)을 통일된 인터페이스로 처리 |

## 6. 하드웨어 전략

### 6.1 Brainless Robot Solution — 하드웨어 경량화

로봇마다 비싼 제어용 PC를 장착하는 대신, 로봇은 **최소한의 기능(AgentX)**만 남기고 고성능 연산은 **클라우드/엣지 서버**가 담당합니다.

| 항목 | 기대 효과 |
|------|----------|
| **하드웨어 원가** | 15~25% 절감 |
| **배터리 효율** | 연산 부하 감소로 운용 시간 증가 |
| **유지보수** | 로봇 측 소프트웨어 업데이트 최소화 |
| **확장성** | 새 로봇 추가 시 서버 측 설정만으로 통합 가능 |

### 6.2 Retrofit — 레거시 장비의 현대화

공장의 **구형 설비(Legacy)**를 버리지 않고 XDT에 연결하는 전략입니다.

- 구형 장비에 **NSS 호환 센서와 제어기(CNC 등)**를 부착
- **지능형 노드로 업그레이드**하여 AI 제어 하에 편입
- 기존 투자를 보호하면서 점진적 스마트화 가능

## 8. 비전 및 확장 가능성

XDT는 단순한 공장 자동화 툴을 넘어, **드론, 휴머노이드, 로봇, 레거시 설비가 공존하는 복잡한 물리 세계를 하나의 지능(AI)으로 통합 제어하는 미래형 인프라**입니다.

### 확장 가능 영역

| 영역 | 적용 분야 |
|------|---------|
| **제조 & 물류** | 스마트 팩토리, 자율 물류센터, 품질 검사 |
| **국방 & 안보** | 무인 정찰, 군집 드론, 전술 시뮬레이션 |
| **스마트 시티** | 도시 인프라, 교통 관제, 에너지 관리 |
| **재난 대응** | 재난 시뮬레이션, 구조 로봇 배치, 대피 최적화 |
| **건설 & 인프라** | 자율 건설, 시설 점검, BIM 연동 |

### 궁극적 목표

> 물리 세계의 모든 것이 디지털 트윈으로 연결되고,
> AI가 그 위에서 생각하고, 판단하고, 실행하는 세상.
> **XDT는 그 세상의 운영체제(OS)입니다.**
